1//! # ACPI (Advanced Configuration and Power Interface) Support
2//!
3//! This crate provides foundational support for parsing and interpreting ACPI
4//! (Advanced Configuration and Power Interface) structures in the kernel. ACPI
5//! is a critical system interface that enables operating systems to discover
6//! and configure hardware components, manage power states, and interact with
7//! platform-specific features.
8//!
9//! ## Overview
10//!
11//! ACPI defines a standardized interface between the operating system and the
12//! platform firmware (BIOS/UEFI). This crate focuses on the early discovery
13//! phase, parsing the fundamental ACPI data structures needed to bootstrap
14//! the full ACPI subsystem.
15//!
16//! ## Architecture
17//!
18//! The ACPI specification defines a hierarchical table structure:
19//!
20//! ```text
21//! UEFI/BIOS Firmware
22//!     ↓
23//! RSDP/XSDP (Root System Description Pointer)
24//!     ↓
25//! RSDT/XSDT (Root/Extended System Description Table)
26//!     ↓
27//! Individual ACPI Tables (FADT, MADT, DSDT, SSDT, etc.)
28//! ```
29//!
30//! ## Key Components
31//!
32//! ### Physical Memory Mapping ([`PhysMapRo`])
33//! Abstract trait for mapping physical memory regions into the kernel's
34//! virtual address space. This allows the ACPI parser to access firmware
35//! tables regardless of the underlying memory management implementation.
36//!
37//! ### RSDP Discovery and Parsing ([`rsdp`])
38//! * **RSDP (ACPI 1.0)**: 20-byte Root System Description Pointer
39//! * **XSDP (ACPI 2.0+)**: Extended version supporting 64-bit addresses
40//! * **Validation**: Checksum verification and signature validation
41//! * **Version Detection**: Automatic handling of ACPI 1.0 vs 2.0+ variants
42//!
43//! ## ACPI Version Support
44//!
45//! ### ACPI 1.0 Support
46//! * **RSDP Structure**: 20-byte legacy format
47//! * **Address Space**: 32-bit physical addresses only
48//! * **RSDT**: Root System Description Table with 32-bit pointers
49//! * **Checksum**: Simple 8-bit checksum of first 20 bytes
50//!
51//! ### ACPI 2.0+ Support
52//! * **XSDP Structure**: Extended format with length field
53//! * **Address Space**: 64-bit physical addresses supported
54//! * **XSDT**: Extended System Description Table with 64-bit pointers
55//! * **Extended Checksum**: Checksum covers entire table structure
56//!
57//! ## Discovery Process
58//!
59//! 1. **Firmware Handoff**: UEFI provides RSDP address in configuration table
60//! 2. **Signature Validation**: Verify "RSD PTR " signature at provided address
61//! 3. **Checksum Verification**: Validate table integrity via checksum
62//! 4. **Version Detection**: Determine ACPI 1.0 vs 2.0+ based on revision field
63//! 5. **Address Extraction**: Extract RSDT/XSDT addresses for further parsing
64//!
65//! ## Memory Management Integration
66//!
67//! The crate uses the [`PhysMapRo`] trait to abstract physical memory access,
68//! enabling integration with various memory management strategies:
69//!
70//! * **Identity Mapping**: Direct physical-to-virtual mapping
71//! * **Higher Half Direct Mapping (HHDM)**: Fixed offset translation
72//! * **Temporary Mapping**: Map-use-unmap pattern for limited virtual space
73//! * **UEFI Runtime Services**: Use firmware-provided mapping functions
74//!
75//! ## Safety Considerations
76//!
77//! ACPI parsing involves extensive unsafe operations due to:
78//!
79//! ### Firmware Interface
80//! * **Untrusted Data**: Firmware-provided addresses and structures
81//! * **Memory Layout**: Platform-specific physical memory organization
82//! * **Packed Structures**: Direct hardware structure interpretation
83//!
84//! ### Validation Strategy
85//! * **Signature Checking**: Verify expected magic bytes in all structures
86//! * **Checksum Validation**: Mathematical integrity verification
87//! * **Bounds Checking**: Validate structure sizes and field ranges
88//! * **Defensive Programming**: Assume firmware data may be malformed
89//!
90//! ## Usage Patterns
91//!
92//! ### Basic RSDP Discovery
93//! ```rust,no_run
94//! use kernel_acpi::{PhysMapRo, rsdp::AcpiRoots};
95//!
96//! struct MyMapper;
97//! impl PhysMapRo for MyMapper {
98//!     unsafe fn map_ro<'a>(&self, paddr: u64, len: usize) -> &'a [u8] {
99//!         // Implementation-specific mapping
100//!         # unimplemented!()
101//!     }
102//! }
103//!
104//! let mapper = MyMapper;
105//! let rsdp_addr = 0x12345000; // From UEFI configuration table
106//!
107//! if let Some(roots) = unsafe { AcpiRoots::parse(&mapper, rsdp_addr) } {
108//!     if let Some(xsdt) = roots.xsdt_addr {
109//!         println!("ACPI 2.0+ detected, XSDT at 0x{:x}", xsdt);
110//!     } else if let Some(rsdt) = roots.rsdt_addr {
111//!         println!("ACPI 1.0 detected, RSDT at 0x{:x}", rsdt);
112//!     }
113//! }
114//! ```
115//!
116//! ## Future Extensions
117//!
118//! This foundational crate enables future ACPI functionality:
119//! * **Table Enumeration**: Parse RSDT/XSDT to discover available tables
120//! * **Specific Parsers**: FADT, MADT, MCFG, and other standard tables
121//! * **AML Interpreter**: ACPI Machine Language execution engine
122//! * **Power Management**: ACPI power state and thermal management
123//! * **Device Enumeration**: PCI Express configuration and device discovery
124//!
125//! ## Standards Compliance
126//!
127//! Implementation follows the ACPI specification:
128//! * **ACPI 1.0**: Legacy 32-bit support for older systems
129//! * **ACPI 2.0+**: Modern 64-bit support with extended features
130//! * **UEFI Integration**: Proper discovery via UEFI configuration tables
131//! * **Checksum Algorithms**: Specification-compliant validation methods
