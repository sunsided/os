1//! # Kernel Initialization and Bootstrap
2//!
3//! This module handles the critical early boot sequence that transforms the kernel
4//! from its initial UEFI-loaded state into a fully functional operating system.
5//! It orchestrates the complex multi-phase initialization process required to
6//! establish a complete runtime environment.
7//!
8//! ## Boot Sequence Overview
9//!
10//! The kernel initialization follows a carefully ordered sequence to safely
11//! transition from UEFI firmware control to autonomous kernel operation:
12//!
13//! 1. **UEFI Handoff** ([`_start_kernel`]) - Naked assembly entry point
14//! 2. **Boot Stack Setup** - Temporary stack for early initialization
15//! 3. **Memory Management** - Physical allocator and virtual memory initialization
16//! 4. **Stack Migration** - Transition to properly allocated kernel stack
17//! 5. **CPU Configuration** - GDT, TSS, IDT, and per-CPU structure setup
18//! 6. **Hardware Initialization** - APIC, timers, and interrupt controllers
19//! 7. **Final Handoff** - Transfer control to main kernel loop
20//!
21//! ## Critical Components
22//!
23//! ### Entry Points
24//! * [`_start_kernel`] - Raw assembly entry point called directly by UEFI loader
25//! * [`kernel_entry_on_boot_stack`] - First Rust code executed on temporary stack
26//! * [`stage_two_init_bootstrap_processor`] - Main initialization on proper kernel stack
27//!
28//! ### Stack Management
29//! * [`BOOT_STACK`] - 64KiB temporary stack in dedicated BSS section
30//! * [`Aligned16`] - 16-byte aligned buffer wrapper for ABI compliance
31//! * [`stage_one_switch_to_stack_and_enter`] - Naked stack switching trampoline
32//!
33//! ### Subsystem Initialization
34//! * [`initialize_memory_management`] - Sets up physical and virtual memory management
35//! * [`initialize_kernel_stack`] - Allocates and maps per-CPU kernel stack
36//! * [`allocate_ist1_stack`] - Creates IST1 stack for exception handling
37//! * [`remap_framebuffer_memory`] - Maps UEFI GOP framebuffer into kernel space
38//!
39//! ## Architecture Details
40//!
41//! ### ABI Compliance
42//! The initialization sequence carefully maintains x86-64 System V ABI requirements:
43//! - 16-byte stack alignment at function entry points
44//! - Proper register usage for parameter passing
45//! - Red-zone avoidance in kernel space
46//!
47//! ### Memory Layout
48//! ```text
49//! UEFI Loader → _start_kernel (naked asm)
50//!                   ↓
51//!               BOOT_STACK (temporary, .bss.boot section)
52//!                   ↓
53//!           kernel_entry_on_boot_stack (early Rust init)
54//!                   ↓
55//!               Kernel Stack (properly allocated via VMM)
56//!                   ↓
57//!       stage_two_init_bootstrap_processor (full initialization)
58//! ```
59//!
60//! ### Privilege and Control Flow
61//! - **Ring 0**: All initialization occurs in kernel mode with full privileges
62//! - **Interrupts**: Disabled during critical initialization phases
63//! - **Virtual Memory**: Higher-half kernel mapped at `0xffffffff80000000`
64//! - **Stack Safety**: Multiple stack transitions with careful alignment management
65//!
66//! ## Safety Considerations
67//!
68//! This module contains extensive unsafe code for:
69//! - Naked assembly functions for low-level CPU control
70//! - Direct memory manipulation and stack switching
71//! - Hardware register programming and MSR access
72//! - Static mutable global state management
73//!
74//! All unsafe operations are carefully isolated and documented, with safety
75//! invariants clearly established for each phase of initialization.
76//!
77//! ## Error Handling
78//!
79//! Early boot failures are generally unrecoverable and result in kernel panic.
80//! The initialization sequence uses `.expect()` patterns to provide meaningful
81//! error messages for debugging boot failures.

124/// Earliest boot stack size. This stack is used only when handing over from UEFI
125/// to the Kernel, and then immediately changed for a properly allocated stack.
126pub const BOOT_STACK_SIZE: usize = 64 * 1024;

128/// A byte buffer with a **guaranteed 16-byte alignment**.
129///
130/// This thin newtype is used to back stacks (or other raw buffers) that must
131/// satisfy the x86-64 SysV ABI alignment rules. In particular, we want the
132/// stack pointer (`RSP`) to be 16-byte aligned at call boundaries and when
133/// entering assembly routines that assume vector-friendly alignment.
134///
135/// - The inner array has length `N`.
136/// - The wrapper enforces `align(16)`, regardless of the platform’s default.
137///
138/// # Why 16 bytes?
139/// Many instructions and calling conventions (e.g., SSE/AVX spills, ABI rules)
140/// expect 16-byte alignment. Some exception/interrupt paths also benefit from a
141/// predictable alignment, so we keep our boot stack aligned from the start.
142///
143/// # Examples
144/// ```
145/// # use core::mem::{align_of, size_of};
146/// # struct Aligned<const N: usize>([u8; N]);
147/// # #[repr(align(16))] struct A<const N: usize>([u8; N]);
148/// # let _ = (size_of::<A<4096>>(), align_of::<A<4096>>());
149/// // 4096 bytes with 16-byte alignment
150/// // let buf = Aligned::<4096>([0; 4096]);
151/// ```
152#[repr(align(16))]
153struct Aligned16<const N: usize>([u8; N]);

155/// Early **boot stack** memory placed in a dedicated BSS section,
156/// used by the boostrap CPU (BSP).
157///
158/// This stack is used before the normal kernel allocator and task stacks exist
159/// (e.g., right after firmware/bootloader hand-off). It is:
160///
161/// - **Static** and zero-initialized (lives in `.bss.boot`).
162/// - **16-byte aligned** via the `Aligned` wrapper.
163/// - **`static mut`** because low-level setup code manipulates it via raw
164///   pointers. Access must be controlled to avoid data races.
165///
166/// Attributes:
167/// - `#[unsafe(link_section = ".bss.boot")]` ensures the symbol is emitted into
168///   a dedicated section (useful for linker scripts, paging attributes, or for
169///   keeping boot-only data together).
170/// - `#[unsafe(no_mangle)]` gives the symbol a stable name so hand-written
171///   assembly (e.g., early entry stubs) can refer to it.
172///
173/// # Safety
174/// - Only one CPU/core should use this stack at a time. Treat it as **single-
175///   owner**, early-boot scratch space.
176/// - Do not use once per-CPU or task stacks are set up.
177/// - Writing/reading it requires `unsafe` because it’s `static mut`.
178#[unsafe(link_section = ".bss.boot")]
179#[unsafe(no_mangle)]
180static mut BOOT_STACK: Aligned16<BOOT_STACK_SIZE> = Aligned16([0; BOOT_STACK_SIZE]);
181
182const _: () = assert!(
183    BOOT_STACK_SIZE.is_multiple_of(16),
184    "BOOT_STACK_SIZE should be 16-byte aligned"
185);

187/// The kernel entry point
188///
189/// # UEFI Interaction
190/// The UEFI loader will jump here after `ExitBootServices`.
191///
192/// # ABI
193/// The ABI is defined as `sysv64` (Rust's `extern "C"`), so the kernel is called
194/// with the `boot_info` pointer in `RDI` (System V AMD64 ABI, as on Linux/x86_64).
195///
196/// # Naked function & Stack
197/// This is a naked function in order to set up the stack ourselves. Without
198/// the `naked` attribute (and the [`naked_asm`](core::arch::naked_asm) instruction), Rust
199/// compiler would apply its own assumptions based on the C ABI and would attempt to
200/// unwind the stack on the call into [`kernel_entry_on_boot_stack`]. Since we're clearing out the stack
201/// here, this would cause UB.
202#[unsafe(no_mangle)]
203#[unsafe(naked)]
204pub extern "C" fn _start_kernel(_boot_info: *const KernelBootInfo) {
205    core::arch::naked_asm!(
206        "cli",
207        // save RDI (boot_info per SysV64)
208        "mov r12, rdi",
209        // Build our own kernel stack and establish a valid call frame for kernel_entry
210        "lea rax, [rip + {stack_sym}]",
211        "add rax, {stack_size}",
212        // Align down to 16
213        "and rax, -16",
214        // Set RSP to the prepared value
215        "mov rsp, rax",
216        // Emulate a CALL by pushing a dummy return address (so RSP % 16 == 8 at entry)
217        "push 0",
218        "xor rbp, rbp",
219        // Restore boot_info into the expected arg register (SysV/C ABI)
220        "mov rdi, r12",
221        // Jump to Rust entry and never return
222        "jmp {rust_entry}",
223        stack_sym = sym BOOT_STACK,
224        stack_size = const BOOT_STACK_SIZE,
225        rust_entry = sym kernel_entry_on_boot_stack,
226    );
227}

229/// Kernel entry running on the boot stack ([`BOOT_STACK`]).
230///
231/// # Notes
232/// * `no_mangle` is used so that [`_start_kernel`] can jump to it by name.
233/// * It uses C ABI to have a defined convention when calling in from ASM.
234/// * The [`_start_kernel`] function keeps `boot_info` in `RDI`, matching C ABI expectations.
235#[unsafe(no_mangle)]
236pub extern "C" fn kernel_entry_on_boot_stack(boot_info: *const KernelBootInfo) -> ! {
237    let logger = QemuLogger::new(LevelFilter::Debug);
238    logger.init().expect("logger init");
239
240    info!("Kernel reporting to QEMU! Initializing bootstrap processor now.");
241    let info = unsafe { CpuidRanges::read() };
242    info!("Running on {}", info.vendor.as_str());
243
244    let bi = unsafe { &*boot_info };
245    trace_boot_info(bi);
246
247    info!("Initializing Virtual Memory Manager ...");
248    initialize_memory_management();
249
250    info!("Initializing Kernel stack ...");
251    let kstack_top = initialize_kernel_stack();
252
253    // Switch to the new stack (align already handled in map_kernel_stack)
254    info!("Switching to boostrap processor kernel stack ...");
255    unsafe {
256        stage_one_switch_to_stack_and_enter(
257            kstack_top,
258            stage_two_init_bootstrap_processor,
259            boot_info,
260        );
261    }
262}

264fn initialize_memory_management() {
265    unsafe {
266        // Initialize the bitmap allocator on the heap.
267        // TODO: Restrict allocator to actual available RAM size.
268        let alloc = init_physical_memory_allocator_once();
269        info!(
270            "Supporting {} MiB of physical RAM",
271            alloc.manageable_size() / 1024 / 1024
272        );
273
274        // Initialize the VMM with the allocator.
275        init_kernel_vmm(HhdmPhysMapper, alloc);
276    }
277}

279fn initialize_kernel_stack() -> KernelStackTop {
280    let kstack_cpu_slot = kstack_slot_for_cpu(0);
281    info!("Designated CPU-specific stack base at {kstack_cpu_slot}.");
282    info!("Allocating bootstrap processor kernel stack ...");
283    let CpuStack {
284        base: _base,
285        top: kstack_top,
286        len: _len,
287    } = try_with_kernel_vmm(FlushTlb::OnSuccess, |vmm| {
288        map_kernel_stack(vmm, kstack_cpu_slot, KERNEL_STACK_SIZE as u64)
289    })
290    .expect("map per-CPU kernel stack");
291
292    info!("Probing new kernel stack at {kstack_top} ...");
293    let probe = (kstack_top.as_u64() - 8) as *mut u64;
294    unsafe {
295        core::ptr::write_volatile(probe, 0xDEAD_BEEF_DEAD_BEEFu64);
296        let _ = core::ptr::read_volatile(probe);
297    }
298
299    kstack_top
300}

302/// Naked jump pad: set RSP and jump; no Rust frame, no locals.
303///
304/// # Safety
305/// This is a **naked jump pad** that switches the active stack pointer (`RSP`)
306/// to a newly-mapped kernel stack and immediately jumps into another function.
307/// It performs no prologue or epilogue and must never return.
308///
309/// ## Calling convention
310/// This function uses the System V AMD64 (Unix x86-64) C ABI.
311/// The caller must pass its arguments exactly as follows:
312///
313/// | Argument | Purpose                  | Register |
314/// |-----------|--------------------------|-----------|
315/// | `top`     | New stack top (RSP target) | `RDI` |
316/// | `entry`   | Function pointer to jump to | `RSI` |
317/// | `boot_info` | Pointer forwarded to `entry` | `RDX` |
318///
319/// The caller places these values in the given registers before the jump.
320/// No arguments are passed on the stack.
321///
322/// ## Operation
323/// ```text
324/// mov rsp, rdi      ; switch to new stack
325/// push 0            ; ensure RSP%16 == 8 for next call per SysV ABI
326/// mov  rdi, rdx     ; move boot_info into first argument register
327/// jmp  rsi          ; tail-jump to entry (never returns)
328/// ```
329///
330/// The `push 0` maintains correct 16-byte stack alignment expected at function
331/// entry. The trampoline then tail-jumps to `entry`, so control never returns
332/// to this function.
333///
334/// ## Notes
335/// * Do **not** mark this with `nostack`; it *does* modify `RSP`.
336/// * The red-zone must be disabled for the kernel target (otherwise writes
337///   below `RSP` may corrupt memory).
338/// * Interrupts should be disabled while switching stacks.
339/// * Because this function is `#[naked]`, you **must not** use any Rust locals
340///   or reference its parameters in Rust code — only inline assembly may appear
341///   in the body.
342///
343/// ## Invariants
344/// * The target stack (`top`) must be fully mapped and writable.
345/// * `entry` must be a valid, non-returning function adhering to the same ABI.
346/// * On success, control never returns; on failure, behavior is undefined.
347#[unsafe(naked)]
348unsafe extern "C" fn stage_one_switch_to_stack_and_enter(
349    _top: VirtualAddress,
350    _entry: extern "C" fn(*const KernelBootInfo, VirtualAddress) -> !,
351    _boot_info: *const KernelBootInfo,
352) -> ! {
353    core::arch::naked_asm!(
354        // rdi=top, rsi=entry, rdx=boot_info
355        "mov r11, rsi", // preserve 'entry' into r11 (caller-saved scratch)
356        "mov rsp, rdi", // set stack = top (rdi still holds top)
357        "push 0",       // align RSP so next call sees RSP%16 == 8
358        "mov rsi, rdi", // rsi = top (second argument for callee)
359        "mov rdi, rdx", // rdi = boot_info (first argument for callee)
360        "jmp r11",      // tail jump to entry; never return
361    )
362}

364/// Per-CPU configuration (GDT, IST, stack, ...)
365///
366/// Currently only one static configuration exists due to early single-core development.
367/// Later on this might be changed into a vector. See [`kstack_slot_for_cpu`] for calculating
368/// per-CPU stack offsets.
369static mut PER_CPU0: PerCpu = PerCpu::new();
370
371extern "C" fn stage_two_init_bootstrap_processor(
372    boot_info: *const KernelBootInfo,
373    kstack_top: KernelStackTop,
374) -> ! {
375    info!("Trampolined onto the kernel stack. Observing kernel stack top at {kstack_top}.");
376    let bi = unsafe { &*boot_info };
377    trace_boot_info(bi);
378
379    info!("Allocating IST1 stack ..");
380    let ist1_top = allocate_ist1_stack();
381
382    // Initialize per-CPU configuration
383    let cpu = initialize_percpu_config_for_bsp(kstack_top, ist1_top);
384
385    info!("Initializing GDT and TSS ...");
386    gdt::init_gdt_and_tss(cpu, kstack_top, ist1_top);
387
388    // Point GS.base to &PerCpu for fast access
389    unsafe {
390        init_gs_bases(cpu);
391    }
392
393    // Enable syscall
394    unsafe {
395        init_syscall(cpu);
396    }
397
398    info!(
399        "Remapping UEFI GOP framebuffer ({size} bytes) ...",
400        size = bi.fb.framebuffer_size
401    );
402    let fb_virt = remap_framebuffer_memory(bi);
403
404    // Initialize the IDT once.
405    info!("Initializing IDT ...");
406
407    unsafe {
408        init_idt_once(Idt::new());
409    }
410
411    // Update the IDT. Enters a critical section and (re-)enables interrupts
412    // when the function returns.
413    info!("Installing interrupt handlers ...");
414    idt_update_in_place(|idt| {
415        idt.init_df_gate_ist(interrupts::df::double_fault_handler, Ist::Ist1); // TODO: Use a different IST from PF
416        idt.init_breakpoint_gate(interrupts::bp::bp_handler);
417        idt.init_syscall_gate();
418        idt.init_ss_fault_gate(interrupts::ss::ss_fault_handler);
419        idt.init_gp_fault_gate(interrupts::gp::gp_fault_handler);
420        idt.init_page_fault_gate_ist(interrupts::page_fault::page_fault_handler, Ist::Ist1);
421        idt.init_timer_gate(interrupts::timer::lapic_timer_handler);
422        idt.init_spurious_interrupt_gate();
423    });
424
425    info!("Estimating TSC frequency ...");
426    let tsc_hz = unsafe { estimate_tsc_hz() };
427    trace_tsc_frequency(tsc_hz);
428
429    // Init LAPIC, store LAPIC ID into per-CPU struct, then arm timer.
430    init_lapic_and_set_cpu_id(cpu);
431    start_lapic_timer(tsc_hz);
432
433    info!("Enabling interrupts ...");
434    sti_enable_interrupts();
435
436    info!("Clearing UEFI pages ...");
437    with_kernel_vmm(|vmm| unsafe { vmm.clear_lower_half() });
438
439    info!("Enabling Supervisor Mode Execution and Access Prevention (SMEP/SMAP)");
440    enable_supervisor_protections();
441
442    info!("Kernel early init is done, jumping into kernel main loop ...");
443    kernel_main(&fb_virt)
444}

446fn enable_supervisor_protections() {
447    unsafe {
448        Cr4::load_unsafe()
449            .with_smep(true)
450            .with_smap(true)
451            .store_unsafe();
452    }
453}

455unsafe fn init_syscall(cpu: &PerCpu) {
456    // Set STAR kernel / user CS bases.
457    unsafe {
458        Ia32Star::from_selectors(&cpu.selectors).store_unsafe();
459    }
460
461    // Set LSTAR to syscall entry stub.
462    let addr = VirtualAddress::from_extern_c_fn(syscall_entry_stub);
463    info!("Syscall entry stubs at {addr}");
464    unsafe {
465        Ia32LStar::new().with_syscall_rip(addr).store_unsafe();
466    }
467
468    // Set FMASK to clear dangerous RFLAGS on syscall entry.
469    unsafe {
470        Ia32Fmask::new_kernel_defaults().store_unsafe();
471    }
472
473    // Enable EFER.SCE (System Call Extensions)
474    unsafe { Efer::load_unsafe().with_sce(true).store_unsafe() }
475}

480fn allocate_ist1_stack() -> Ist1StackTop {
481    let (ist1_base, ist1_top) = try_with_kernel_vmm(FlushTlb::OnSuccess, |vmm| {
482        let slot = ist_slot_for_cpu(0, Ist::Ist1);
483        map_ist_stack(vmm, slot, IST1_SIZE)
484    })
485    .expect("map IST1");
486    info!("IST1 mapped: base={ist1_base}, top={ist1_top}");
487    ist1_top
488}

490fn initialize_percpu_config_for_bsp(
491    kstack_top: KernelStackTop,
492    ist1_top: Ist1StackTop,
493) -> &'static mut PerCpu {
494    #[allow(static_mut_refs)]
495    let p = unsafe { &mut PER_CPU0 };
496    p.cpu_id = 0;
497    p.apic_id = 0; // will be set below by the APIC initialization.
498    p.kstack_top = kstack_top;
499    if let Some(idx) = Ist::Ist1.tss_index() {
500        p.ist_stacks[idx] = ist1_top;
501    }
502    p
503}

513/// Remaps the boot framebuffer memory into the kernel's virtual address space.
514///
515/// UEFI provides the physical address of the framebuffer in the boot info, but does not
516/// include it in the memory mapping table. This means the kernel must manually map the
517/// framebuffer into its own virtual address space to access it. This function sets up the
518/// necessary mapping so the framebuffer can be used by the kernel.
519fn remap_framebuffer_memory(bi: &KernelBootInfo) -> FramebufferInfo {
520    // Map framebuffer
521    let fb_pa = PhysicalAddress::new(bi.fb.framebuffer_ptr);
522    let fb_len = bi.fb.framebuffer_size;
523    let va_base = HHDM_BASE + VGA_LIKE_OFFSET;
524    let fb_flags = VirtualMemoryPageBits::default()
525        .with_writable(true)
526        .with_write_combining()
527        .with_global(true)
528        .with_no_execute(true);
529
530    try_with_kernel_vmm(FlushTlb::OnSuccess, |vmm| {
531        vmm.map_region(
532            AllocationTarget::Kernel,
533            va_base,
534            fb_pa,
535            fb_len,
536            fb_flags,
537            fb_flags,
538        )
539    })
540    .expect("Framebuffer mapping failed");
541
542    // Return updated FramebufferInfo with new virtual address
543    let mut fb_virt = bi.fb.clone();
544    fb_virt.framebuffer_ptr = (va_base + (fb_pa.as_u64() & 0xFFF)).as_u64(); // preserve offset within page
545    info!("Remapped frame buffer to {va_base}");
546    fb_virt
547}
