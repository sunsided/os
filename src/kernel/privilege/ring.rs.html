1/// CPU privilege rings, represented as numerical privilege levels (0–3).
2///
3/// These are the **Descriptor Privilege Levels (DPLs)** and **Current Privilege Level (CPL)**
4/// values used by the x86 and x86-64 architectures to enforce access control.
5///
6/// ## Overview
7/// x86 defines four privilege rings (also called protection levels):
8///
9/// | Ring | Numeric Level | Typical Use | Privilege |
10/// |------|----------------|--------------|------------|
11/// | **Ring 0** | 0 | Kernel / supervisor mode | Highest privilege (can execute all instructions) |
12/// | **Ring 1** | 1 | Drivers / OS subsystems (rarely used) | Intermediate privilege |
13/// | **Ring 2** | 2 | Drivers / OS subsystems (rarely used) | Intermediate privilege |
14/// | **Ring 3** | 3 | User-mode applications | Lowest privilege (restricted instructions) |
15///
16/// In most modern 64-bit kernels, only **Ring 0** (kernel) and **Ring 3** (user) are used.
17/// Rings 1 and 2 exist in the architecture but are conventionally unused.
18///
19/// ## Relationship to DPL, CPL, and RPL
20/// - **CPL (Current Privilege Level):** the ring number of the currently executing code segment.
21/// - **DPL (Descriptor Privilege Level):** the ring required to *access* a descriptor (segment,
22///   gate, etc.). A gate with `DPL=3` can be called from user space (`CPL=3`), whereas `DPL=0`
23///   requires kernel mode.
24/// - **RPL (Requested Privilege Level):** stored in the low two bits of a segment selector,
25///   allowing software to request access as if from a higher ring.
26///
27/// When the CPU checks access permissions, it effectively compares
28/// `max(CPL, RPL) <= DPL`. If the condition fails, a `#GP` (General Protection Fault) is raised.
29///
30/// ## Usage
31/// This enum is mainly used when configuring descriptor privilege levels in
32/// structures such as the [Interrupt Descriptor Table (IDT)](crate::interrupts::Idt):
33///
34/// ```ignore
35/// idt[0x80]
36///     .set_handler(syscall_int80_handler)
37///     .selector(KERNEL_CS)
38///     .dpl_ring(Ring::Ring3)  // allow user mode to invoke this gate
39///     .present(true)
40///     .gate_interrupt();
41/// ```
42///
43/// ## Safety note
44/// - Setting an incorrect ring level (e.g., allowing user access to a privileged handler)
45///   can compromise system isolation and security.
46/// - Use `Ring::Ring3` only for well-defined syscall or user interrupt entry points.
47///
48/// For most kernel code, only [`Ring::Ring0`] and [`Ring::Ring3`] are relevant.
49///
50/// See also: Intel SDM Vol. 3A, §5.5 “Privilege Levels”.
51#[derive(Copy, Clone, Eq, PartialEq, Debug)]
52#[repr(u8)]
53pub enum Ring {
54    /// **Ring 0 (DPL = 0)** — Kernel or supervisor mode.
55    ///
56    /// This is the most privileged level. Code running in Ring 0 can execute
57    /// any instruction, access any memory region, and perform I/O operations.
58    /// Typically used for the operating system kernel and core interrupt handlers.
59    Ring0 = 0,
60
61    /// **Ring 1 (DPL = 1)** — Historically for OS components or drivers.
62    ///
63    /// Rarely used on modern systems. Present for architectural completeness.
64    #[deprecated]
65    Ring1 = 1,
66
67    /// **Ring 2 (DPL = 2)** — Historically for OS subsystems or drivers.
68    ///
69    /// Like Ring 1, almost never used in 64-bit mode; modern kernels consolidate
70    /// all privileged code in Ring 0.
71    #[deprecated]
72    Ring2 = 2,
73
74    /// **Ring 3 (DPL = 3)** — User-mode applications.
75    ///
76    /// The least privileged level. Code running in Ring 3 cannot perform I/O or
77    /// privileged instructions and can only access user-mapped memory.
78    /// Typical for userland processes and syscall entry points.
79    Ring3 = 3,
80}

82impl Ring {
83    #[inline]
84    #[must_use]
85    pub const fn to_u8(self) -> u8 {
86        self as u8
87    }
88}

90impl From<Ring> for u8 {
91    #[inline]
92    fn from(r: Ring) -> Self {
93        r.to_u8()
94    }
95}

97impl TryFrom<u8> for Ring {
98    type Error = u8;
99
100    #[inline]
101    fn try_from(r: u8) -> Result<Self, Self::Error> {
102        if r <= 3 {
103            Ok(unsafe { core::mem::transmute::<u8, Self>(r) })
104        } else {
105            Err(r)
106        }
107    }
