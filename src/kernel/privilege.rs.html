1//! # Privilege Levels (x86-64)
2//!
3//! This module defines the **privilege level hierarchy** used by the CPU to
4//! isolate protection domains (e.g. kernel vs. user mode).
5//!
6//! ## Overview
7//!
8//! x86-64 implements four hierarchical *rings* (`0–3`), of which only two are
9//! normally used in long mode:
10//!
11//! - **Ring 0** — kernel / supervisor mode (highest privilege)
12//! - **Ring 3** — user mode (least privilege)
13//!
14//! These levels appear in three distinct contexts that the CPU enforces rules
15//! between:
16//!
17//! | Concept | Stored in | Purpose |
18//! |----------|------------|----------|
19//! | [`Ring`] | the current `CS` selector | the **current privilege level** (`CPL`) |
20//! | [`Rpl`]  | the low 2 bits of a selector | the **requested privilege level** from the requester |
21//! | [`Dpl`]  | bits 45–46 of a descriptor | the **descriptor privilege level** of the target |
22//!
23//! The CPU performs access checks such as:
24//!
25//! - **Data segment load:** `max(CPL, RPL) ≤ DPL`
26//! - **Non-conforming code:** `CPL == DPL`
27//! - **Conforming code:** `CPL ≤ DPL`
28//! - **Stack segment (long mode):** `CPL == RPL == DPL`
29//!
30//! ## Structure
31//!
32//! This module re-exports:
33//!
34//! - [`Ring`] — enumeration of privilege rings (`Ring0` … `Ring3`).
35//! - [`Rpl`] — requested privilege level plus helpers for selectors.
36//! - [`Dpl`] — descriptor privilege level plus access-check predicates.
37//! - [`KERNEL_RPL`](rpl::KERNEL_RPL)/[`USER_RPL`](rpl::USER_RPL) — constants for common use.
38//!
39//! Together, these types provide a clear, type-safe way to reason about
40//! privilege levels in descriptor and selector code, such as when building a
41//! GDT, TSS, or when transitioning between user and kernel mode.
42//!
43//! ## Example
44//!
45//! ```rust
46//! use arch_x86_64::privilege::{Ring, Rpl, Dpl, USER_RPL};
47//!
48//! let cpl = Ring::Ring0; // current level
49//! let rpl = USER_RPL;    // selector carries Ring3
50//! let dpl = Dpl::Ring3;  // descriptor is user-accessible
51//!
52//! assert!(dpl.permits_data_load(cpl, rpl));
53//! ```
54//!
55//! ## References
56//! - Intel® SDM, Vol 3A: *Protection; Descriptor Privilege Level (DPL)*
57//! - AMD64 Architecture Programmer’s Manual, Vol 2: *System Programming*
