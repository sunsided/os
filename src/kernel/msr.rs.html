1//! # Model-Specific Registers (MSR) utilities
2//!
3//! This module provides low-level access to CPU **Model-Specific Registers (MSRs)**,
4//! particularly those related to segment base addresses used in 64-bit mode.
5//!
6//! ## Background
7//! The x86-64 architecture provides the **GS** and **FS** segment registers as
8//! general-purpose base registers for thread-local or per-CPU data. Their actual
9//! 64-bit base addresses are stored in model-specific registers, accessible via
10//! the privileged `RDMSR` and `WRMSR` instructions.
11//!
12//! Commonly used MSRs include:
13//! - `IA32_GS_BASE` (`0xC000_0101)`: current GS base address used by `mov %gs:...`
14//! - `IA32_KERNEL_GS_BASE` (`0xC000_0102)`: swap value used when executing `swapgs`
15//!
16//! The kernel typically uses `IA32_KERNEL_GS_BASE` to store the **kernel-side GS base**
17//! (e.g., a pointer to per-CPU data), while userland may use `IA32_GS_BASE` for TLS.
18//!
19//! On a transition from user to kernel (e.g., via `syscall` or interrupt), the CPU
20//! executes `swapgs`, which exchanges the contents of `IA32_GS_BASE` and
21//! `IA32_KERNEL_GS_BASE`, giving the kernel immediate access to its per-CPU data.
22//!
23//! ## References
24//! - Intel SDM Vol. 3, §2.5.4 “FS and GS Base Address Registers”
25//! - AMD64 Architecture Programmer’s Manual Vol. 2, §4.8.3 “MSRs for FS/GS Base”
26

37/// Convenience: initialize both GS bases to the same per-CPU pointer
38/// (common during early boot so a later `swapgs` is a no-op).
39///
40/// # Safety
41/// Same as [`set_gs_base`](Ia32GsBaseMsr::set_gs_base) and [`set_kernel_gs_base`](Ia32KernelGsBaseMsr::set_kernel_gs_base).
42#[inline]
43pub unsafe fn init_gs_bases(percpu: &PerCpu) {
44    unsafe {
45        <Ia32GsBaseMsr as Ia32GsBaseMsrExt>::set_gs_base(percpu);
46        <Ia32KernelGsBaseMsr as Ia32KernelGsBaseMsrExt>::set_kernel_gs_base(percpu); // for SWAPGS
47    }
