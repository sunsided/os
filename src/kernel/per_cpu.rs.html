1//! # Per-CPU Data Structures and Management
2//!
3//! This module provides the foundation for per-CPU data storage and management
4//! in the kernel, enabling efficient CPU-specific state tracking and future
5//! symmetric multiprocessing (SMP) support. It centralizes all CPU-local
6//! resources including stacks, descriptors, and runtime state.
7//!
8//! ## Architecture Overview
9//!
10//! The per-CPU system is built around the [`PerCpu`] structure, which serves as
11//! the central repository for all CPU-specific state. This design enables:
12//!
13//! * **Cache-Friendly Access**: 64-byte alignment prevents false sharing between CPUs
14//! * **Fast Retrieval**: GS-base MSR points directly to current CPU's [`PerCpu`] instance
15//! * **Scalable Design**: Virtual memory layout supports multiple CPUs with dedicated regions
16//! * **Stack Safety**: Guard pages and separate stack types prevent overflow corruption
17//!
18//! ## Key Components
19//!
20//! ### Core Per-CPU Structure ([`PerCpu`])
21//! * **CPU Identification**: Logical CPU ID and Local APIC ID tracking
22//! * **Task Management**: Current task pointer for future scheduling support
23//! * **Hardware Descriptors**: TSS (Task State Segment), GDT, and selector storage
24//! * **Stack Management**: Kernel stack and IST (Interrupt Stack Table) pointers
25//! * **Accounting**: Tick counter and scratch space for CPU-specific data
26//!
27//! ### Stack Management Subsystem
28//! * [`kernel_stacks`]: Virtual memory layout for per-CPU kernel stacks
29//! * [`ist_stacks`]: IST stack allocation for exception handling
30//! * [`stack`]: Common stack mapping and guard page implementation
31//!
32//! ## Virtual Memory Layout
33//!
34//! The module defines separate virtual memory regions for different stack types:
35//!
36//! ### Kernel Stacks (`0xffff_ff00_0000_0000` base)
37//! ```text
38//! CPU 0: [ Guard 4K ][ Stack up to 1M ]
39//! CPU 1: [ Guard 4K ][ Stack up to 1M ]  ← +1MB stride
40//! CPU 2: [ Guard 4K ][ Stack up to 1M ]  ← +2MB stride
41//! ```
42//!
43//! ### IST Stacks (`0xffff_ff10_0000_0000` base)
44//! ```text
45//! CPU 0:
46//!   IST1: [ Guard 4K ][ Stack 128K ]
47//!   IST2: [ Guard 4K ][ Stack 128K ]  ← +128K stride per IST
48//!   ...
49//! CPU 1: (same layout at +1MB offset)
50//! ```
51//!
52//! ## Stack Safety Features
53//!
54//! * **Guard Pages**: Unmapped 4KiB pages below each stack catch overflow via page fault
55//! * **ABI Alignment**: All stack tops maintain 16-byte alignment for x86-64 System V ABI
56//! * **Isolation**: Separate virtual regions prevent stack type confusion
57//! * **Size Limits**: Configurable maximum sizes prevent excessive memory usage
58//!
59//! ## Fast Access Pattern
60//!
61//! The [`PerCpu::current()`] function provides constant-time access to the current
62//! CPU's data structure:
63//!
64//! ```rust
65//! let cpu = PerCpu::current();
66//! cpu.ticks.fetch_add(1, Ordering::Relaxed);
67//! ```
68//!
69//! This relies on the GS-base MSR pointing to the current CPU's [`PerCpu`] instance,
70//! set up during CPU initialization.
71//!
72//! ## Future SMP Considerations
73//!
74//! While currently supporting only the Bootstrap Processor (BSP), the design
75//! accommodates future Application Processor (AP) support:
76//!
77//! * **Scalable Addressing**: Virtual memory layout supports arbitrary CPU counts
78//! * **Cache Line Alignment**: 64-byte alignment prevents false sharing
79//! * **Independent State**: Each CPU maintains completely separate data structures
80//! * **Atomic Operations**: Tick counters and task pointers use atomic primitives
81//!
82//! ## Safety and Concurrency
83//!
84//! * **Single Owner**: Each [`PerCpu`] instance belongs to exactly one CPU
85//! * **Atomic Fields**: Shared state uses appropriate atomic primitives
86//! * **Memory Safety**: Guard pages and bounds checking prevent corruption
87//! * **Interrupt Safety**: Access patterns work correctly during interrupt handling

99#[repr(C, align(64))] // avoid false sharing; nice for future SMP
100pub struct PerCpu {
101    /// Logical CPU index (0..n-1). Often equals BSP/AP numbering.
102    pub cpu_id: u32,
103
104    /// APIC/LAPIC id if you add AP startup later.
105    pub apic_id: u32,
106
107    /// Pointer to the current running task (kernel struct). Optional for now.
108    pub current_task: core::sync::atomic::AtomicPtr<Task>,
109
110    /// 64-bit TSS required in long mode (rsp0, `ISTx`, iopb).
111    pub tss: Tss64,
112
113    /// Kernel stack top used on CPL3→CPL0 transitions (loaded from TSS.rsp0).
114    pub kstack_top: VirtualAddress,
115
116    /// Interrupt Stack Table entries (alternate hard stacks, e.g., NMI/#DF).
117    ///
118    /// **Slot 0:** [`IST1`](super::interrupts::Ist::Ist1)
119    pub ist_stacks: [VirtualAddress; 7],
120
121    /// GDT storage
122    pub gdt: Gdt,
123
124    /// GDT selector values (CS/DS/…/TSS).
125    pub selectors: Selectors,
126
127    /// Quick per-CPU scratch / small allocator caches if you add them later.
128    pub scratch: PerCpuScratch,
129
130    /// Accounting / stats you might grow.
131    pub ticks: core::sync::atomic::AtomicU64,
132}
133
134pub struct Task;
135pub struct PerCpuScratch;
136
137impl PerCpu {
138    pub const fn new() -> Self {
139        Self {
140            cpu_id: 0,
141            apic_id: 0,
142            current_task: core::sync::atomic::AtomicPtr::new(core::ptr::null_mut()),
143            tss: Tss64::new(),
144            kstack_top: VirtualAddress::zero(),
145            ist_stacks: [VirtualAddress::zero(); 7],
146            gdt: Gdt::new(),
147            selectors: Selectors::new(),
148            scratch: PerCpuScratch,
149            ticks: core::sync::atomic::AtomicU64::new(0),
150        }
151    }
152
153    pub const fn tss_ptr(&self) -> *const Tss64 {
154        core::ptr::from_ref::<Tss64>(&self.tss)
155    }
156
157    pub const fn tss_base(&self) -> VirtualAddress {
158        VirtualAddress::from_ptr(self.tss_ptr())
159    }
160
161    #[inline(always)]
162    #[allow(clippy::inline_always)]
163    pub unsafe fn current() -> &'static Self {
164        unsafe { <Ia32GsBaseMsr as Ia32GsBaseMsrExt>::current() }
165    }
166}
