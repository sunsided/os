1//! # Interrupt Descriptor Table (IDT)
2//!
3//! The **IDT** tells the CPU which handler to call for each **interrupt**, **exception**,
4//! or **software trap**. In x86-64, the IDT is an array of 16-byte gate descriptors,
5//! and a pointer to it is stored in the **IDTR** (loaded via `lidt`).
6//!
7//! ## How the IDT relates to the GDT/TSS
8//! Although most *segmentation* is off in long mode, privilege transitions on
9//! interrupts still consult state provided by your **GDT/TSS** setup:
10//!
11//! - On a transition from Ring-3 → Ring-0 (e.g., a syscall interrupt gate with DPL=3,
12//!   or a user fault), the CPU **loads the stack pointer from the TSS** (`rsp0`).
13//!   This requires that you have installed a valid TSS descriptor in the GDT and
14//!   executed `ltr` once. See [`crate::gdt::init_gdt_and_tss`] and [`crate::gdt::TSS_SEL`].
15//! - If an IDT entry uses an **IST** (Interrupt Stack Table) slot, the CPU will switch
16//!   to the corresponding `istN` stack from the **TSS** before entering the handler.
17//!   This is how you can make double-fault or NMI handlers robust even if the
18//!   current stack is corrupted. See [`Tss64`](super::tss::Tss64) and the TSS initialization.
19//!
20//! > In short: **load the GDT and TSS first**, then `lidt`, then enable interrupts.
21//!
22//! ## What this module provides
23//! - A single, **global** IDT storage (`static mut IDT`) and helpers to:
24//!   - Install it once into the IDTR: [`init_idt_once`]
25//!   - Mutate it safely in place without reloading IDTR: [`idt_update_in_place`]
26//!   - Borrow a mutable reference when you know what you’re doing: [`idt_mut`]
27//!
28//! The pattern is intentionally simple for a bootstrap CPU. If we later move to a
29//! per-CPU IDT, we can keep the same call sites and swap the backing storage.
30//!
31//! ## Quick start
32//! ```no_run
33//! use crate::interrupts::Idt;
34//!
35//! //Build an IDT with your stubs/gates.
36//! let mut idt = Idt::new();
37//! idt.init_exceptions();            // your helper
38//! idt.init_irq_gates();             // your helper
39//! idt.init_syscall_gate(int80_stub); // example
40//!
41//! // Make sure GDT+TSS are installed once (stack for ring changes / IST).
42//! //    See: crate::gdt::init_gdt_and_tss
43//!
44//! // Load the IDT into IDTR.
45//! unsafe { crate::idt::init_idt_once(idt) };
46//!
47//! // Enable interrupts when ready.
48//! unsafe { core::arch::asm!("sti") };
49//! ```
50//!
51//! ## Safety & concurrency model
52//! - **Install once per CPU** before enabling interrupts. This module currently holds
53//!   a single global IDT (shared by CPUs). If multiple cores update entries, serialize
54//!   with your chosen mechanism (IPI + stop-the-world, spinlocks, etc.).
55//! - [`idt_update_in_place`] wraps updates in `cli`/`sti` on the **local** CPU and
56//!   issues a full memory fence to make changes visible before interrupts resume.
57//! - You **do not** need to execute `lidt` again when editing entries **in place**,
58//!   as long as the base address and limit of the table remain unchanged.
59//!
60//! ## Ordering checklist (typical bootstrap)
61//! 1. Map memory and enter long mode.
62//! 2. **GDT/TSS:** call [`crate::gdt::init_gdt_and_tss`] (loads GDT via `lgdt`, sets up TSS, executes `ltr`).
63//! 3. **IDT:** build an `Idt`, then call [`init_idt_once`] (loads IDTR via `lidt`).
64//! 4. Configure PIC/APIC as needed, then `sti`.
65//!
66//! With this sequence, user→kernel transitions will get a sane Ring-0 stack via
67//! TSS.`rsp0`, and critical handlers can use IST stacks if you configured them.
68

74/// The global interrupt descriptor table.
75static mut IDT: MaybeUninit<Idt> = MaybeUninit::uninit();

77/// Initialize the **global Interrupt Descriptor Table (IDT)** once and load it into the CPU.
78///
79/// # Overview
80/// This function permanently installs the given [`Idt`] as the system-wide interrupt table
81/// by:
82/// 1. Writing it into the global static storage `IDT`, and
83/// 2. Executing the `lidt` instruction to load its base and limit into the CPU’s **IDTR**.
84///
85/// After this call, the CPU will consult the given table whenever an interrupt,
86/// exception, or software trap occurs.
87///
88/// # Safety
89/// - Must be called **exactly once per CPU** before any interrupts are enabled.
90/// - `idt` must be a fully initialized and valid interrupt table whose memory remains
91///   **permanently allocated** (e.g. in static storage).
92/// - This function is `unsafe` because it alters the CPU’s global interrupt state.
93///   Incorrect usage may lead to undefined behavior if the IDT points to invalid handlers.
94///
95/// # Notes
96/// - The `lidt` instruction only loads a pointer; the CPU fetches entries from memory
97///   dynamically. You may therefore modify entries in place later without re-executing
98///   `lidt`, provided the table base and size remain unchanged.
99pub unsafe fn init_idt_once(idt: Idt) {
100    #[allow(static_mut_refs)]
101    unsafe {
102        IDT.write(idt);
103        IDT.assume_init_ref().load();
104    }
105}
106
107/// Borrow a mutable reference to the **global IDT**.
108///
109/// # Safety
110/// - The global IDT must have been previously initialized with [`init_idt_once`].
111/// - The returned reference has `'static` lifetime and must not be aliased mutably
112///   from multiple threads or CPUs simultaneously without synchronization.
113/// - You typically use this in short critical sections where interrupts are disabled.
114///
115/// # Example
116/// ```no_run
117/// unsafe {
118///     let idt = idt_mut();
119///     idt.set_handler(0x80, syscall_entry);
120/// }
121/// ```
122unsafe fn idt_mut() -> &'static mut Idt {
123    #[allow(static_mut_refs)]
124    unsafe {
125        IDT.assume_init_mut()
126    }
127}
128
129/// Update entries of the global IDT **in place**, without reloading `lidt`.
130///
131/// # Behavior
132/// - Disables interrupts locally (`cli`) to prevent concurrent entry during mutation.
133/// - Applies the user-supplied closure `f(&mut Idt)`.
134/// - Executes a full memory fence to ensure visibility of updates before re-enabling interrupts.
135/// - Re-enables interrupts (`sti`) after modification.
136///
137/// This is the preferred mechanism for dynamically patching interrupt vectors (e.g.
138/// installing a new syscall handler or replacing an exception stub) when the IDT base
139/// address and limit remain unchanged.
140///
141/// # Safety & Concurrency
142/// - Because the IDT is shared between all CPUs, ensure that concurrent updates from
143///   other cores are serialized appropriately (e.g. via IPI, spinlock, or stop-the-world).
144/// - Updates to the same IDT entry must be atomic with respect to interrupt delivery.
145/// - Avoid modifying critical vectors (NMI, double-fault) while they are active.
146///
147/// # Example
148/// ```no_run
149/// // Swap syscall handler at runtime.
150/// idt_update_in_place(|idt| {
151///     idt.init_syscall_gate(new_int80_entry);
152/// });
153/// ```
154pub fn idt_update_in_place<F: FnOnce(&mut Idt)>(f: F) {
155    #[allow(static_mut_refs)]
156    unsafe {
157        debug_assert!(IDT.assume_init_ref().is_loaded(), "IDT is not installed");
158    }
159
160    let _guard = IrqGuard::new();
161    unsafe {
162        let idt = idt_mut();
163        f(idt);
164
165        // Ensure the write completes before re-enabling interrupts.
166        atomic::fence(atomic::Ordering::SeqCst);
167    }
168}
