1//! # QEMU Development and Debug Support
2//!
3//! This crate provides specialized debugging and development utilities for kernels
4//! running under the QEMU emulator. It enables efficient logging, tracing, and
5//! diagnostic output that significantly simplifies kernel development and debugging
6//! in virtualized environments.
7//!
8//! ## Overview
9//!
10//! Kernel development presents unique debugging challenges: no standard I/O,
11//! limited debugging tools, and the need for early-boot diagnostics. This crate
12//! addresses these challenges by leveraging QEMU's built-in debugging features,
13//! particularly the debug port mechanism for host-side output.
14//!
15//! ## Key Features
16//!
17//! * **Zero-Allocation Logging**: Efficient output without dynamic memory allocation
18//! * **Standard Log Integration**: Compatible with Rust's `log` crate ecosystem
19//! * **Feature-Gated Operation**: Can be completely disabled for release builds
20//! * **UTF-8 Support**: Proper Unicode character encoding for debug output
21//! * **Early Boot Support**: Works before full kernel initialization
22//!
23//! ## Architecture
24//!
25//! ### QEMU Debug Port Interface
26//! The crate communicates with QEMU through a dedicated I/O port:
27//! * **Port Address**: `0x402` (QEMU's standard debug port)
28//! * **Protocol**: Simple byte-by-byte character transmission
29//! * **Direction**: Kernel-to-host output only
30//! * **Performance**: Extremely fast, no buffering required
31//!
32//! ### Output Mechanism
33//! ```text
34//! Kernel Code
35//!     ↓
36//! qemu_trace! macro
37//!     ↓
38//! QemuSink (fmt::Write)
39//!     ↓
40//! dbg_putc() → I/O port 0x402
41//!     ↓
42//! QEMU Debug Output
43//!     ↓
44//! Host Terminal/Console
45//! ```
46//!
47//! ## Core Components
48//!
49//! ### QEMU Logger ([`QemuLogger`])
50//! A `log::Log` implementation that routes log messages to QEMU's debug port:
51//! * **Level Filtering**: Configurable log level thresholds
52//! * **Target Support**: Module and component-specific logging
53//! * **Static Initialization**: No-allocation setup for kernel environments
54//! * **Format Control**: Structured log message formatting
55//!
56//! ### Trace Macro ([`qemu_trace!`])
57//! Direct debug output bypassing the logging framework:
58//! * **Zero-Overhead**: Compile-time format string processing
59//! * **UTF-8 Encoding**: Proper multi-byte character support
60//! * **Feature Gating**: Completely removed when `enabled` feature is disabled
61//! * **Format Compatibility**: Supports `format!`-style argument patterns
62//!
63//! ### Output Sink ([`qemu_fmt::QemuSink`])
64//! A `core::fmt::Write` implementation for structured output:
65//! * **Character-by-Character**: Immediate output with no buffering
66//! * **UTF-8 Encoding**: Proper Unicode character decomposition
67//! * **Error Resilience**: Best-effort output with graceful degradation
68//!
69//! ## Feature System
70//!
71//! The crate uses Cargo features to control compilation and runtime behavior:
72//!
73//! ### `enabled` Feature (default)
74//! When enabled:
75//! * Full QEMU debug port functionality
76//! * Active trace macro implementation
77//! * I/O port operations compiled in
78//! * UTF-8 encoding and formatting support
79//!
80//! When disabled:
81//! * All debug operations become no-ops
82//! * Zero runtime overhead
83//! * No I/O port access
84//! * Suitable for production builds
85//!
86//! ## Usage Patterns
87//!
88//! ### Basic Logging Setup
89//! ```rust,no_run
90//! use kernel_qemu::QemuLogger;
91//! use log::{LevelFilter, info};
92//!
93//! // Early in kernel initialization
94//! let logger = QemuLogger::new(LevelFilter::Debug);
95//! logger.init().expect("logger initialization");
96//!
97//! // Standard logging throughout kernel
98//! info!("Kernel subsystem initialized");
99//! ```
100//!
101//! ### Direct Trace Output
102//! ```rust,ignore
103//! use kernel_qemu::qemu_trace;
104//!
105//! // Immediate debug output
106//! qemu_trace!("Debug value: {:#x}\n", register_value);
107//!
108//! // Complex formatting
109//! qemu_trace!("CPU {}: Exception {} at RIP={:#x}\n",
110//!             cpu_id, exception_number, instruction_pointer);
111//! ```
112//!
113//! ### Conditional Compilation
114//! ```rust
115//! #[cfg(feature = "qemu-debug")]
116//! use kernel_qemu::qemu_trace;
117//!
118//! fn debug_memory_layout() {
119//!     #[cfg(feature = "qemu-debug")]
120//!     qemu_trace!("Memory layout: base={:#x} size={:#x}\n", base, size);
121//! }
122//! ```
123//!
124//! ## QEMU Integration
125//!
126//! ### Host-Side Configuration
127//! To capture kernel debug output on the host:
128//! ```bash
129//! # Standard QEMU invocation with debug output
130//! qemu-system-x86_64 -kernel kernel.bin -debugcon stdio
131//!
132//! # Redirect to file
133//! qemu-system-x86_64 -kernel kernel.bin -debugcon file:debug.log
134//!
135//! # Network debugging (advanced)
136//! qemu-system-x86_64 -kernel kernel.bin -debugcon tcp:127.0.0.1:1234,server
137//! ```
138//!
139//! ### Debug Port Mechanism
140//! QEMU's debug console feature (`-debugcon`) captures writes to port `0x402`:
141//! * **Real-time Output**: Immediate visibility of kernel messages
142//! * **No Guest Impact**: Zero performance impact on kernel execution
143//! * **Flexible Routing**: Output can go to stdio, files, or network
144//! * **Cross-Platform**: Works on all QEMU-supported host platforms
145//!
146//! ## Performance Characteristics
147//!
148//! * **Macro Overhead**: Zero when `enabled` feature is disabled
149//! * **I/O Latency**: Sub-microsecond output latency in QEMU
150//! * **Memory Usage**: No dynamic allocation, minimal static footprint
151//! * **CPU Impact**: Negligible processing overhead
152//!
153//! ## Safety Considerations
154//!
155//! ### I/O Port Access
156//! * **Privilege Requirements**: Requires kernel-mode execution (Ring 0)
157//! * **Port Safety**: QEMU debug port is read-only from host perspective
158//! * **Hardware Compatibility**: Safe no-op on real hardware (port typically unused)
159//! * **Error Handling**: Graceful degradation if QEMU features unavailable
160//!
161//! ### UTF-8 Encoding
162//! * **Memory Safety**: No buffer overflows in character encoding
163//! * **Encoding Validity**: Proper UTF-8 sequence generation
164//! * **Error Recovery**: Invalid characters handled gracefully
165//!
166//! ## Development Workflow
167//!
168//! This crate is essential for iterative kernel development:
169//!
170//! 1. **Early Debugging**: Debug boot process and initialization
171//! 2. **Runtime Diagnostics**: Monitor kernel operation and state changes
172//! 3. **Performance Analysis**: Trace timing-critical operations
173//! 4. **Error Investigation**: Capture crash information and stack traces
174//! 5. **Feature Development**: Validate new subsystem implementation
175//!
176//! The integration with QEMU makes it an indispensable tool for productive
177//! kernel development, providing the debugging visibility essential for
178//! complex systems programming.

187#[cfg(feature = "enabled")]
188#[doc(hidden)]
189pub mod qemu_fmt {
190    use core::fmt::{self, Write};
191
192    /// The port number for QEMU's debug port.
193    const QEMU_DEBUG_PORT: u16 = 0x402;
194
195    /// Write a single character to QEMU's debug port.
196    #[allow(clippy::inline_always)]
197    #[inline(always)]
198    pub fn dbg_putc(c: u8) {
199        unsafe { outb(QEMU_DEBUG_PORT, c) }
200    }
201
202    /// Write to QEMU's port.
203    #[allow(clippy::inline_always)]
204    #[inline(always)]
205    unsafe fn outb(port: u16, val: u8) {
206        unsafe {
207            core::arch::asm!(
208            "out dx, al",
209            in("dx") port,
210            in("al") val,
211            options(nomem, preserves_flags)
212            );
213        }
214    }
215
216    // TODO: Model this as an actual sink for arbitrary port write
217    pub struct QemuSink;
218
219    impl Write for QemuSink {
220        #[inline]
221        fn write_str(&mut self, s: &str) -> fmt::Result {
222            for b in s.bytes() {
223                dbg_putc(b);
224            }
225            Ok(())
226        }
227
228        #[inline]
229        fn write_char(&mut self, c: char) -> fmt::Result {
230            // UTF-8 encode without allocation.
231            let mut buf = [0u8; 4];
232            let s = c.encode_utf8(&mut buf);
233            self.write_str(s)
234        }
235    }
236
237    #[doc(hidden)]
238    #[inline(always)]
239    #[allow(clippy::inline_always)]
240    pub fn qemu_write(args: fmt::Arguments) {
241        // Ignore errors; this is best-effort debug output.
242        let _ = fmt::write(&mut QemuSink, args);
243    }
244}
