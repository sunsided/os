1//! # `kernel_sync` â€” kernel-friendly synchronization primitives
2//!
3//! Minimal `no_std` locks and cells designed for kernels and bare-metal systems.
4//! Primitives aim to be small, predictable, and explicit about safety and
5//! memory-ordering. No poisoning, no allocation, and no OS dependencies.
6//!
7//! ## Modules & types
8//! - [`SpinLock`]/[`SpinLockGuard`]: TATAS spinlock for a single value.
9//! - [`RawSpin`], [`RawTicket`]: raw, low-level lock primitives.
10//! - [`Mutex<T, R>`]/[`MutexGuard`]: generic RAII mutex over any raw lock `R`.
11//! - [`SpinMutex<T>`], [`TicketMutex<T>`]: convenient mutex aliases.
12//! - [`IrqGuard`], [`IrqMutex`]: scope-based interrupt disable + mutex guard
13//!   (`x86/x86_64`, privileged mode).
14//! - [`SyncOnceCell<T>`]: single-writer, multi-reader, spin-based once-cell.
15//!
16//! ## Concurrency model
17//! These primitives rely on acquire/release atomics and CPU-local spinning.
18//! They are intended for **short** critical sections under **low contention**.
19//! On SMP, some operations (e.g., TLB invalidation) act on the **local** CPU;
20//! coordinate cross-CPU work as needed.
21//!
22//! ## Safety
23//! Many APIs are safe to *call* but have **correctness requirements** (e.g.,
24//! only use `invlpg` for the current address space; interrupts must be legal
25//! to disable in your context). Unsafe methods clearly document their preconditions.
26//!
27//! ## `no_std`
28//! This crate is `no_std` by default and uses inline assembly on `x86/x86_64`
29//! for interrupt control where applicable.
30

48/// A `Mutex` backed by a [`RawSpin`] lock.
49///
50/// Fairness: none. Prefer under light contention.
51pub type SpinMutex<T> = Mutex<T, RawSpin>;
52
53/// A `Mutex` backed by a [`RawTicket`] lock.
54///
55/// Fairness: FIFO. Slightly higher latency, avoids starvation.
56pub type TicketMutex<T> = Mutex<T, RawTicket>;

58impl<T> SpinMutex<T> {
59    /// Creates a new [`SpinMutex`] containing `value`.
60    ///
61    /// # Examples
62    ///
63    /// ```
64    /// use kernel_sync::SpinMutex;
65    /// let m = SpinMutex::new(0u32);
66    /// *m.lock() = 1;
67    /// ```

88/// A low-level lock interface used by [`Mutex`].
89///
90/// Implementors provide the raw operations; higher-level types (like [`Mutex`])
91/// add RAII and safe access to protected data.
92///
93/// ### Semantics
94/// - `raw_lock` must provide mutual exclusion and publish the critical section
95///   with **acquire** semantics for the caller.
96/// - `raw_try_lock` attempts to acquire without blocking.
97///
98/// Implementations may spin, sleep, or otherwise block, depending on context.
99pub trait RawLock {
100    /// Acquires the lock (may spin or block) and returns when the caller holds it.
101    fn raw_lock(&self);
102
103    /// Attempts to acquire the lock without blocking.
104    ///
105    /// Returns `true` on success, `false` if the lock is currently held.
106    fn raw_try_lock(&self) -> bool;
107}
108
109/// A companion trait that releases a raw lock.
110///
111/// # Safety
112/// The caller must ensure the current execution context **holds** the lock
113/// being unlocked. Violating this precondition can corrupt memory or break
114/// invariants.
115///
116/// Implementations must:
117/// - Use **release** semantics to publish writes from the critical section.
118/// - Leave the lock in a state that another `raw_lock`/`raw_try_lock` can acquire.
119pub trait RawUnlock {
120    /// Releases the lock previously acquired by this context.
121    ///
122    /// # Safety
123    /// Caller must currently hold the lock.
124    unsafe fn raw_unlock(&self);
125}
