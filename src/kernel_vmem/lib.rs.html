1//! # Virtual Memory Support
2//!
3//! Minimal x86-64 paging helpers for a hobby OS loader/kernel.
4//!
5//! ## What you get
6//! - An [`address space`](address_space) describing a `PML4` root page table.
7//! - Tiny [`PhysicalAddress`]/[`VirtualAddress`](kernel_memory_addresses::VirtualAddress) newtypes (u64) to avoid mixing address kinds.
8//! - A [`PageSize`](kernel_memory_addresses::PageSize) enum for 4 KiB / 2 MiB / 1 GiB mappings.
9//! - x86-64 page-table [`VirtualMemoryPageBits`] with practical explanations.
10//! - A 4 KiB-aligned [`PageTable`] wrapper and index helpers.
11//! - A tiny allocator/mapper interface ([`PhysFrameAlloc`], [`PhysMapper`]).
12//!
13//! ## x86-64 Virtual Address → Physical Address Walk
14//!
15//! Each 48-bit virtual address is divided into five fields:
16//!
17//! ```text
18//! | 47‒39 | 38‒30 | 29‒21 | 20‒12 | 11‒0   |
19//! |  PML4 |  PDPT |   PD  |   PT  | Offset |
20//! ```
21//!
22//! The CPU uses these fields as **indices** into four levels of page tables,
23//! each level containing 512 (2⁹) entries of 8 bytes (64 bits) each.
24//!
25//! ```text
26//!  PML4  →  PDPT  →  PD  →  PT  →  Physical Page
27//!   │        │        │        │
28//!   │        │        │        └───► PTE   (Page Table Entry)  → maps 4 KiB page
29//!   │        │        └────────────► PDE   (Page Directory Entry) → PS=1 → 2 MiB page
30//!   │        └─────────────────────► PDPTE (Page Directory Pointer Table Entry) → PS=1 → 1 GiB page
31//!   └──────────────────────────────► PML4E (Page Map Level 4 Entry)
32//! ```
33//!
34//! ### Levels and their roles
35//!
36//! | Level | Table name | Entry name | Description |
37//! |:------|:------------|:-----------|:-------------|
38//! | 1 | **PML4** (Page Map Level 4) | **PML4E** | Top-level table; each entry points to a PDPT. One PML4 table per address space, referenced by Control Register 3 ([`CR3`](https://wiki.osdev.org/CPU_Registers_x86#CR3)). |
39//! | 2 | **PDPT** (Page Directory Pointer Table) | **PDPTE** | Each entry points to a PD. If `PS=1`, it directly maps a 1 GiB page (leaf). |
40//! | 3 | **PD** (Page Directory) | **PDE** | Each entry points to a PT. If `PS=1`, it directly maps a 2 MiB page (leaf). |
41//! | 4 | **PT** (Page Table) | **PTE** | Each entry maps a 4 KiB physical page (always a leaf). |
42//!
43//! ### Leaf vs. non-leaf entries
44//!
45//! - A **leaf entry** directly maps physical memory — it contains the physical base address
46//!   and the permission bits ([`PRESENT`](VirtualMemoryPageBits::with_present), [`WRITABLE`](VirtualMemoryPageBits::with_writable), [`USER`](VirtualMemoryPageBits::with_user), [`GLOBAL`](VirtualMemoryPageBits::with_global), [`NX`](VirtualMemoryPageBits::with_no_execute), etc.).
47//!   - A **PTE** is always a leaf (maps 4 KiB).
48//!   - A **PDE** with `PS=1` is a leaf (maps 2 MiB).
49//!   - A **PDPTE** with `PS=1` is a leaf (maps 1 GiB).
50//!
51//! - A **non-leaf entry** points to the next lower table level and continues the walk.
52//!   For example, a PML4E points to a PDPT, and a PDE with `PS=0` points to a PT.
53//!
54//! ### Offset
55//!
56//! - The final **Offset** field (bits 11–0) selects the byte inside the 4 KiB (or larger) page.
57//!
58//! ### Summary
59//!
60//! A canonical 48-bit virtual address is effectively:
61//!
62//! ```text
63//! VA = [PML4:9] [PDPT:9] [PD:9] [PT:9] [Offset:12]
64//! ```
65//!
66//! This creates a four-level translation tree that can map up to **256 TiB** of
67//! virtual address space, using leaf pages of 1 GiB, 2 MiB, or 4 KiB depending
68//! on which level the translation stops.

89/// Minimal allocator that hands out **4 KiB** page-table frames.
90pub trait PhysFrameAlloc {
91    /// Allocate a zeroed 4 KiB page suitable for a page-table.
92    fn alloc_4k(&mut self) -> Option<PhysicalPage<Size4K>>;
93
94    /// Deallocate a zeroed 4 KiB page suitable for a page-table.
95    fn free_4k(&mut self, pa: PhysicalPage<Size4K>);
96}

98/// Mapper capable of temporarily viewing physical frames as typed tables.
99pub trait PhysMapper {
100    /// Map a 4 KiB physical frame and get a **mutable** reference to type `T`.
101    ///
102    /// # Safety
103    /// The implementation must ensure that the returned reference aliases the
104    /// mapped frame, and that writes reach memory.
105    #[allow(clippy::mut_from_ref)]
106    unsafe fn phys_to_mut<T>(&self, at: PhysicalAddress) -> &mut T;
107}

218/// Reads the current value of the **CR3 register** (the page table base register)
219/// and returns the physical address of the top-level page table (PML4).
220///
221/// # Safety
222/// This function is **unsafe** because it directly accesses a CPU control register.
223/// It must only be called in privileged (ring 0) code where paging is active and
224/// the CR3 contents are valid. Calling it from user mode or before enabling paging
225/// will cause undefined behavior.
226///
227/// # Details
228/// - On x86-64, CR3 holds the **physical base address** of the currently active
229///   PML4 (Page Map Level 4) table.
230/// - The low 12 bits of CR3 contain **flags** (e.g., PCID, reserved bits),
231///   so this function masks them out to obtain a 4 KiB-aligned physical address.
232/// - The returned address represents the root of the current virtual memory
233///   hierarchy used for address translation.
234///
235/// # Returns
236/// The 4 KiB-aligned [`PhysicalAddress`] of the current PML4 table.
237///
238/// # Example
239/// ```no_run
240/// use kernel_vmem::read_cr3_phys;
241///
242/// // SAFETY: Only call from kernel mode with paging enabled.
243/// let current_pml4 = unsafe { read_cr3_phys() };
244/// println!("Active PML4 base: {:#x}", current_pml4.as_u64());
245/// ```
246#[allow(clippy::inline_always)]
247#[inline(always)]
248#[must_use]
249pub unsafe fn read_cr3_phys() -> PhysicalAddress {
250    let cr3 = unsafe { Cr3::load_unsafe() };
251    let phys_base = cr3.pml4_phys();
252
253    // Clear PCID / low bits by turning it into a 4K page base and back to an address.
254    let page = phys_base.page::<Size4K>(); // drop low 12 bits
255    debug_assert_eq!(
256        phys_base,
257        page.base(),
258        "PML4 phys base is not 4 KiB aligned"
259    );
260
261    phys_base
262}

264/// Invalidate one page in the TLB on the **current CPU** for the **current** address space.
265///
266/// Emits `invlpg [va]`.
267///
268/// # Safety
269/// The caller must uphold:
270/// - **Privilege:** Executed in a context where `invlpg` is permitted (e.g., ring 0).
271/// - **Canonical VA:** `page.base()` must be canonical for the current paging mode.
272/// - **Current address space:** Only use after modifying the **active** address space
273///   (same CR3/PCID as the caller). It does not invalidate entries for other CR3s.
274/// - **Scope:** It affects **only** the local CPU. Coordinate remote invalidations
275///   on other CPUs that could have cached the translation.
276/// - **PCID/global pages:** If PCID or global mappings are in use, ensure this
277///   instruction’s semantics match your needs; otherwise prefer INVPCID or a
278///   global-flush strategy when required.
279#[inline]
280pub unsafe fn invalidate_tlb_page(page: VirtualPage<Size4K>) {
281    let va = page.base().as_u64();
282    unsafe {
283        core::arch::asm!(
284            "invlpg [{}]",
285            in(reg) va,
286            options(nostack, preserves_flags),
287        );
288    }
