Module ist_stacks 

Source
Expand description

Virtual layout for IST stacks: per-CPU × per-IST slots with a guard page.

Layout (virtual): IST_BASE + cpu_id * CPU_STRIDE + (ist_idx-1) * IST_SLOT_STRIDE -> [ guard (4 KiB, unmapped) ][ IST stack bytes, RW|NX ]

Notes

  • We keep IST stacks in a separate region from kernel stacks to simplify debugging and avoid tight packing constraints.
  • One unmapped 4 KiB guard below each IST stack catches overflows.
  • IST_SLOT_STRIDE must be >= guard + max IST size you’ll map.

Constants§

IST1_SIZE
16 KiB is enough for #PF/#DF handlers.
IST_BASE
Virtual base for all IST stacks (choose a disjoint, canonical kernel range). Picked in the higher half, far from your kernel-stack region (0xffff_ff00_…).
IST_CPU_STRIDE
Per-CPU stride in the IST region (bytes). Leave ample room for 7 ISTs.
IST_GUARD
4 KiB guard below each IST stack.
IST_SLOTS_PER_CPU
Number of hardware IST entries supported by x86_64 TSS.
IST_SLOT_STRIDE
Per-IST stride inside one CPU’s IST area (bytes). With 7 ISTs × 128 KiB < 1 MiB, this fits comfortably.

ist_slot_for_cpu
Return the guard page base for (cpu_id, ist_idx) (1..=7). The first mapped byte (stack base) is guard + 4 KiB.
ist_top_for
Convenience: compute the 16-byte–aligned top given a mapped length. You’ll typically map [guard (4 KiB)][ist_bytes] and then use this top.
max_ist_bytes
Maximum usable IST bytes per slot (excludes the 4 KiB guard).
