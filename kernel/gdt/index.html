Â§Global Descriptor Table (GDT) & Task State Segment (TSS) descriptor wiring for long mode

In 64-bit mode (â€œlong modeâ€), classic segmentation for code/data is largely disabled, but segment selectors still matter:

  • They distinguish code vs. data/stack segments.
  • They carry the Descriptor Privilege Level (DPL) used by the CPU to enforce privilege transitions (Ring-0 â†” Ring-3).
  • They locate the Task State Segment (TSS) through a 16-byte system descriptor in the GDT so the CPU can fetch rsp0 (kernel stack pointer) and IST stacks on privilege changes and fault handling.

  • Returning to userland with iretq requires user code/data descriptors in the GDT and using their selectors (with RPL=3) in the iret frame.
  • Reliable userâ†’kernel transitions require a loaded TSS (via ltr), because the CPU uses tss.rsp0 as the Ring-0 stack on entry.

This module builds a typed GDT, loads it with lgdt, refreshes the data segments, and loads the Task Register (ltr) with the TSS selector.

Â§SMP / per-CPU note

The TSS is per-CPU. If you run multiple CPUs, allocate a TSS (and typically a GDT) per CPU and call the init routine on each CPU during bring-up.

Â§Preconditions

  • Long mode and paging are enabled.
  • The GDT (and TSS) memory is mapped and writable during setup and readable thereafter at the virtual addresses you pass in.
  • Interrupts should be masked during the switch to avoid using half-set state.
