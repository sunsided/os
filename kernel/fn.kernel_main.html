Main kernel loop, running with all memory (including framebuffer) properly mapped.

ยงEntry point

UEFI enters the kernel in _start_kernel, from where we initialize the boot stack, set up memory management and then jump here.

ยงMemory Safety

At this point, the kernel operates with virtual addresses set up by the VMM, and the framebuffer is accessible at its mapped virtual address. All further kernel code should use these mapped addresses, not physical ones.
