Â§kernel_sync â€” kernel-friendly synchronization primitives

Minimal no_std locks and cells designed for kernels and bare-metal systems. Primitives aim to be small, predictable, and explicit about safety and memory-ordering. No poisoning, no allocation, and no OS dependencies.

Â§Modules & types

  • SpinLock/SpinLockGuard: TATAS spinlock for a single value.
  • RawSpin, RawTicket: raw, low-level lock primitives.
  • Mutex<T, R>/MutexGuard: generic RAII mutex over any raw lock R.
  • SpinMutex<T>, TicketMutex<T>: convenient mutex aliases.
  • IrqGuard, IrqMutex: scope-based interrupt disable + mutex guard (x86/x86_64, privileged mode).
  • SyncOnceCell<T>: single-writer, multi-reader, spin-based once-cell.

Â§Concurrency model

These primitives rely on acquire/release atomics and CPU-local spinning. They are intended for short critical sections under low contention. On SMP, some operations (e.g., TLB invalidation) act on the local CPU; coordinate cross-CPU work as needed.

Â§Safety

Many APIs are safe to call but have correctness requirements (e.g., only use invlpg for the current address space; interrupts must be legal to disable in your context). Unsafe methods clearly document their preconditions.

Â§no_std

This crate is no_std by default and uses inline assembly on x86/x86_64 for interrupt control where applicable.
